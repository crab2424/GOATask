# GOATask 修正・機能追加設計

## 決定事項

- フォルダ／プロジェクト削除時の所属先は、削除確認時に選択できるようにする。
  - 親へ移動（ルートの場合はルート）
  - 未分類へ移動
- メモのフォルダ一括保存は `.md` / `.txt` / `.zip` を選択可能にする。
  - コンテキストメニュー内のサブメニュー（PCはホバー、タッチ端末はタップ）で表示する。
- 共有リンクの有効期限は7日を標準値とする。
- 共有ファイルは削除操作時に即時削除する。共有リンクの期限切れはリンクを無効化し、元ファイルは別途削除する。
- Oracleの容量・インスタンス仕様の確定は後回しにする。

## 実装方針

1. ナビゲーションのルート／空白部分にコンテキストメニューを追加する。タスク画面ではタスク・プロジェクト作成、メモ画面ではメモ・フォルダ作成を表示する。
2. タスク編集カード外のpointerdownで編集をキャンセルする。フォーム内部、ダイアログ、コンテキストメニューは除外する。
3. 単語テストは回答ごとに統計APIへ保存し、モード切替・更新・タブ終了で未保存にならないようにする。中断時の一括保存との二重保存を避ける。
4. 一括保存は、現在フォルダのみ／子フォルダを含む全体を選べる設計にする。個別メモの既存エクスポート形式を再利用する。
5. ファイル共有は非公開Object Storage＋期限付き署名URLを基本とする。DBにはファイルメタデータと共有リンクの期限・失効状態を保存する。容量上限と1ファイル上限はOracle仕様確認後に確定する。

## 削除挙動の注意

現状バックエンドは、子フォルダ／子プロジェクトを親へ繰り上げ、直下メモ／タスクをNULL（未分類）にしている。UIの説明とは一致していないため、選択した所属先をAPIで受け取る設計に変更する。

## Oracle確認箇所

OCIコンソールでは、対象インスタンスの「詳細」だけでなく、次を確認する。

- インスタンス：シェイプ、OCPU、メモリ、ブート・ボリューム、リージョン、可用性ドメイン
- ストレージ：Object Storageのバケット、ストレージ層、使用量、残容量
- テナンシー：ガバナンスと管理 > 制限、割当て、使用量
- 課金：コスト分析、無料枠／Always Freeの使用状況

画像にあるリージョン `ap-tokyo-1`、可用性ドメイン `AD-1`、Canonical Ubuntu 24.04、稼働中であることは確認できるが、OCPU・メモリ・ブートボリューム容量・Object Storage使用量は画像だけでは確定できない。
